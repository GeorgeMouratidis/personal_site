UX/UI Project

Project Description

In this project the goal was to review and redesign the web page of a greek bank, based on the principles of UX/UI design. The first part of the project consists of an expert based evaluation, in which a heuristic evaluation was used. For this part we depended on the 10 rules of evaluation from Nielsen, and for each problem spotted in the website, a different level of severity was assigned. In addition, we reviewed the website using the Keystroke Level Model (KLM), a technique with which we measure the actual time needed to complete certain tasks.

The second part of the project consists of the redesign of some of the most problematic features of the website. For this reason, we made use of the Figma software, which allowed us to design continuous paths from a certain frame of the page to another. For the design, it was important to create representative characters from different groups (usually called Personas in UX/UI), as well as the context in which they would use the website (usually called Scenarios). After establishing the right personas and scenarios, the design's main target was for the user to accomplish some main tasks as effectively as possible.
